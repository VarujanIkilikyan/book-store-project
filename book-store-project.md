
# 📚 Mini Project: Book Store with Virtual Currency & PDF Access

**Курс:** Node.js & Express.js (с EJS и файловым хранилищем)  
**Уровень:** Intermediate  
**Цель:** Построить веб-приложение, где пользователи могут покупать книги на виртуальные монеты и скачивать PDF-файлы только тех книг, которые купили.

---

## 🎯 Обзор

Ты создаёшь **интернет-магазин книг**, где:
- У каждого пользователя есть **1000 монет** при регистрации
- В магазине есть **общая библиотека книг**, каждая с ценой
- Пользователь может **покупать книги**, и после покупки — **скачивать PDF**
- Доступ к файлу есть **только у купивших**
- Всё хранится в JSON-файлах, файлы — в папке `public/books/pdf/`
- Фронтенд на **EJS**, бэкенд на **Express.js**

## ✅ Функциональные требования

### 1. Регистрация и вход
- При регистрации: `username`, `email`, `password`, `age`
- Пароль хешируется (`md5(md5(pass) + SECRET)`)
- Новый пользователь получает **1000 монет**
- Вход → токен (через `CryptoJS`), сохраняется в cookie или `req.session`

### 2. Библиотека книг
- `books.json` содержит:
  ```json
  {
    "id": "uuid",
    "title": "The Node.js Guide",
    "author": "Jane Doe",
    "price": 250,
    "pdfPath": "/books/pdf/nodejs-guide.pdf",
    "description": "..."
  }

  Папка public/books/pdf/ содержит PDF-файлы

Прямой доступ к /books/pdf/... должен быть запрещён

3. Покупка книги
Пользователь видит список книг с ценой

Кнопка “Купить” → POST /purchases

Логика:

Проверить, есть ли у пользователя достаточно монет

Если да — списать монеты, создать запись в purchases.json

Обновить баланс в users.json

Нельзя купить одну и ту же книгу дважды

4. Мои книги
Страница /my-books — только купленные книги

Каждая книга — с кнопкой “Скачать PDF”

При клике → /download/:bookId → проверка через middleware → отдача файла

5. Защита файлов
Создай middleware fileAccess.js, который:

Проверяет req.userId

Ищет в purchases.json, купил ли он эту книгу

Если да → res.download(pdfPath)

Если нет → 403 Forbidden

❗ Никаких прямых ссылок вроде /public/books/pdf/book.pdf. Только через защищённый роут.

6. Интерфейс (EJS)
Главная: приветствие, кнопки “Войти”, “Регистрация”

Личный кабинет: баланс, список всех книг, кнопка “Купить”

Мои книги: список купленных, кнопка “Скачать”

Все страницы — с навигацией и стилями

🔗 Ключевые роуты
Метод	Путь	Описание
GET	/	Главная (приветствие)
GET	/auth/login	Форма входа
POST	/auth/login	Вход
GET	/books	Все книги (с ценой и кнопкой “Купить”)
POST	/purchases	Покупка книги
GET	/my-books	Список купленных книг
GET	/download/:bookId	Скачивание PDF (только для купивших)
GET	/logout	Выход


🧠 Сложные моменты (где можно ошибиться)
Прямой доступ к PDF
→ Запрети через Express:

js
app.use('/books/pdf', (req, res) => {
  res.status(403).send('Access denied');
});
Скачивание файла
→ Используй res.download(path) — он безопаснее res.sendFile

Баланс при перезагрузке
→ Все изменения баланса должны сохраняться в users.json

Повторная покупка
→ Перед покупкой проверяй purchases.json, не куплена ли книга
